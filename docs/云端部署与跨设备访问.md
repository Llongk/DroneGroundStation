# 云端部署与跨设备访问

## 1. 文档目标

本文说明如何把 DroneGroundStation 的手机与 STM32 遥测从 Qt 桌面端上传到云端，并在另一台电脑、手机或平板上查看实时数据和 D1 历史数据。

当前系统由三部分组成：

```text
手机浏览器 / STM32
        │ HTTP 或 TCP/JSON
        ▼
Qt DroneGroundStation
        ├─ SQLite：本地历史与离线上传队列
        ├─ MQTT/TLS：发布实时消息到 EMQX Cloud
        └─ HTTPS：直接写入云端 /api/ingest
                         │
                         ▼
             Cloudflare D1 telemetry_records
                         │
                         ▼
             cloud-dashboard 跨设备展示
```

实时数据与历史数据是两条互补链路：

- 实时链路：浏览器使用 WSS 连接 EMQX，订阅 `dgs/#`。
- 历史链路：Qt 使用 HTTPS 写入 D1，网页通过 `/api/history` 读取。
- 离线保护：HTTPS 失败时记录保留在 SQLite `cloud_outbox`，重启后继续补传。
- 调度策略：最新实时记录优先，空闲时间继续按顺序补传旧记录。

## 3. 桌面端云配置

复制配置模板：

```powershell
Copy-Item config/cloud.example.json config/cloud.json
```

编辑 `config/cloud.json`：

| 字段 | 说明 |
|---|---|
| `host` | EMQX 部署概览中的连接地址，不带协议和端口 |
| `port` | MQTT over TLS 使用 `8883` |
| `username` | EMQX 客户端认证用户名 |
| `password` | EMQX 客户端认证密码 |
| `clientId` | 当前电脑唯一客户端编号 |
| `enabled` | 是否启用 MQTT 实时发布 |
| `historyIngestUrl` | 云端网页的 `/api/ingest` 完整 HTTPS 地址 |
| `historyIngestKey` | 与云端 `INGEST_API_KEY` 完全一致的随机密钥 |

安全要求：

- `config/cloud.json` 已被 `.gitignore` 排除，不得强制提交。
- 不要在 QML、C++、README、截图或工单中写入真实密码。
- `historyIngestKey` 泄露后应立即在云端轮换，并更新上传端配置。

## 4. EMQX Cloud 配置

1. 创建 EMQX Serverless 部署。
2. 在“访问控制 → 客户端认证”添加用户名和强密码。
3. 在“客户端授权”允许该用户发布和订阅 `dgs/#`。
4. 桌面端使用 MQTT/TLS 端口 `8883`。
5. 网页端使用 WebSocket over TLS 端口 `8084`。
6. 从部署概览下载 CA 文件，保存为 `config/emqxsl-ca.crt`。

主要主题：

```text
dgs/UAV_01/phone/telemetry
dgs/UAV_01/stm32/telemetry
```

可以在 EMQX“诊断工具 → 在线调试”中订阅 `dgs/#`，验证实时 JSON 是否到达 Broker。

## 5. 云端网页工程

网页源码位于 `cloud-dashboard/`，技术栈如下：

- React 19、Next 16、TypeScript
- vinext、Cloudflare Worker
- Cloudflare D1、Drizzle ORM
- MQTT.js，通过 WSS 订阅 EMQX

本地运行：

```powershell
cd cloud-dashboard
npm install
npm run dev
```

质量检查：

```powershell
npm run lint
npm test
npm run build
```

部署时必须绑定：

- D1 绑定名：`DB`
- Secret：`INGEST_API_KEY`

`INGEST_API_KEY` 只存在云端环境变量中，不写入 Git。部署完成后，把公开站点地址加上 `/api/ingest` 写入桌面端 `historyIngestUrl`。

## 6. HTTP API

### 6.1 写入遥测

```http
POST /api/ingest
Content-Type: application/json
X-Ingest-Key: <secret>
```

请求体：

```json
{
  "topic": "dgs/UAV_01/stm32/telemetry",
  "clientid": "DGS-PC-01",
  "qos": 1,
  "timestamp": 1784261492078,
  "payload": {
    "message_id": "unique-message-id",
    "source": "stm32",
    "latitude": 40.104165,
    "longitude": 116.357812,
    "battery": 80
  }
}
```

返回值：

```json
{ "ok": true, "messageId": "unique-message-id" }
```

保护措施：

- 密钥不匹配返回 `401`。
- 只允许 `dgs/` 开头的主题。
- 请求体上限为 128 KiB。
- `message_id` 唯一索引保证重复补传不会重复入库。

### 6.2 查询历史

```http
GET /api/history?source=stm32&limit=500
GET /api/history?source=phone&limit=500
GET /api/history?source=stm32&limit=500&before=<毫秒时间戳>
```

- `source` 仅区分 `phone` 和 `stm32`。
- `limit` 范围为 1～1000。
- `before` 用于向前分页。

## 7. D1 数据模型

表名：`telemetry_records`。

| 字段 | 用途 |
|---|---|
| `message_id` | 全局去重标识 |
| `topic` | 原始 MQTT 主题 |
| `source` | `phone` 或 `stm32` |
| `device_id`、`session_id` | 设备与登录会话 |
| `sequence` | STM32 序号 |
| `received_at` | 云端接收时间 |
| `latitude`、`longitude` | 位置 |
| `altitude`、`speed`、`heading` | 飞行状态 |
| `battery` | 电量百分比 |
| `temperature`、`humidity` | 环境数据 |
| `alarm_code` | 设备告警代码 |
| `payload` | 完整原始 JSON |

索引：

- `telemetry_message_id_uq`：消息去重。
- `telemetry_source_time_idx`：按来源和时间读取曲线。
- `telemetry_session_idx`：按飞行会话查询。

## 8. 跨设备查看

仅查看云端数据时，另一台设备不需要 Qt、证书、MQTT 密码或上传密钥，只需要：

1. 能连接互联网。
2. 使用现代浏览器打开部署后的 HTTPS 站点。
3. 切换“手机 / STM32”数据源。
4. 点击“读取云端历史”加载 D1 曲线。
5. 若要看实时消息，在页面输入只具备 `dgs/#` 订阅权限的 EMQX 查看账号。

建议为网页查看端单独创建只读 MQTT 用户，禁止发布主题。

## 9. 迁移与恢复

### 9.1 仅迁移查看端

无需复制任何文件，直接访问公网 HTTPS 地址。

### 9.2 迁移 Qt 上传端

需要复制或重新生成：

- 打包后的 Qt 程序及 Qt 运行库。
- `config/cloud.json`。
- `config/emqxsl-ca.crt`。
- 如需继承本地历史，复制 `database/FlightHistory.db`。

关闭程序后再复制数据库，同时复制 `.db-wal`、`.db-shm`，或先让 SQLite 完成检查点。首次运行没有数据库时会自动建库建表。

### 9.3 重新部署网页

1. 克隆仓库并进入 `cloud-dashboard/`。
2. 安装 Node.js 22.13 或更高版本。
3. 安装依赖并执行测试。
4. 创建/绑定 D1 数据库。
5. 设置新的 `INGEST_API_KEY`。
6. 部署并更新 Qt 的 `historyIngestUrl` 与 `historyIngestKey`。

## 10. 常见故障

### 手机连接成功但云端没有数据

- “手机已连接”只表示页面访问了 Qt，不等于浏览器已取得 GPS。
- 检查手机页面是否显示 GPS 获取成功和上传成功。
- 确认已登录 Qt；未开始历史会话时不会写入云队列。
- 查看 `cloud_outbox` 是否存在 `phone` 主题记录。
- 新版本会优先上传最新记录，旧版本可能被 STM32 历史积压阻塞。

### EMQX 显示连接数但 TPS 为 0

- 检查 Qt 日志中是否出现“EMQX 云端已连接”。
- 在线调试订阅 `dgs/#`。
- 核对授权规则是否允许发布对应主题。

### D1 始终是 0 条

- 检查 `historyIngestUrl` 是否包含 `/api/ingest`。
- 检查 Qt 与云端 `INGEST_API_KEY` 是否一致。
- 确认电脑网线能访问互联网，Wi-Fi 到 STM32 只负责局域网遥测。
- 查看 Qt 日志中的 `D1 history upload failed`、HTTP 状态码和响应内容。

### 网页只有历史、没有实时数据

- D1 历史与 EMQX 实时是两条链路。
- 输入 EMQX WSS 地址、只读用户名和密码后点击连接。
- 确认端口 `8084` 和 `dgs/#` 订阅权限。

## 11. Git 提交规范

提交前执行：

```powershell
git status --short
git diff --check
git grep -n -I -E "historyIngestKey|password|INGEST_API_KEY" -- . ":!config/cloud.example.json" ":!docs"
```

严禁提交：

- `config/cloud.json`
- `.env*`
- `database/*.db*`
- `node_modules/`、`.wrangler/`、`.vinext/`、`dist/`
- 私钥、访问令牌、真实 MQTT 密码

推荐提交顺序：

```powershell
git add .
git diff --cached --check
git commit -m "feat: add cloud telemetry dashboard and documentation"
git push origin main
git push github main
```
