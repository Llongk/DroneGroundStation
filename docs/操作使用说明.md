# DroneGroundStation 详细操作与开发说明

## 1. 项目目标

本项目将两种现场数据采集方式统一到一个 Qt 地面站中：

1. 手机作为传感器端，通过浏览器采集 GPS、姿态、海拔、电量和温湿度，以 HTTP/JSON 上传到 Qt。
2. STM32/ESP8266 作为 AP 和 TCP 服务端，Qt 作为 TCP 客户端接收遥测并下发飞控命令。

手机数据与 STM32 数据始终保持独立：使用不同的状态属性、地图轨迹和 SQLite 数据表。为防止两个来源同时更新界面，同一时刻只允许一个活动设备。

## 2. 总体结构与数据流

```text
手机浏览器
  ├─ GET / 或 /index.html ───────────────┐
  ├─ POST /gps ──────────────────────────┤
  └─ POST /sensor ───────────────────────┤
                                          ▼
                                    Backend (HTTP :8080)
                                      │ GPS信号 │ 温湿度信号
                                      ▼         ▼
                                  主页面      SensorBackend
                                      │         │
                                      └────┬────┘
                                           ▼
                                    HistoryDatabase
                                      phone_data

STM32 AP/TCP服务端 (:8080)
  ├─ 遥测 JSON + CRLF ────────────────► SensorBackend ─► STM32页面/地图/曲线
  └─ 命令 ACK  ◄──────────────────────── 飞控命令 JSON + CRLF
                                               │
                                               ▼
                                        HistoryDatabase
                                          stm32_data
```

核心设计：

- C++ 负责网络、JSON 校验、设备状态和数据库；QML 负责展示与交互。
- STM32 JSON 帧以 `\r\n` 结束，TCP 缓冲区支持粘包与拆包。
- 手机活动后暂停 STM32 探测；手机 8 秒没有有效请求后恢复 STM32 探测。
- STM32 命令带独立 `command_seq`，只有序号匹配的 ACK 才完成当前命令。

## 3. 环境与构建

| 项目 | 建议版本/说明 |
|---|---|
| 操作系统 | Windows 10/11 64 位 |
| Qt | Qt 6.11.1 |
| 编译器 | MSVC 2022 64 位 |
| CMake | 3.16+ |
| C++ | C++17 |

Qt 模块：Quick、QuickControls2、Network、Sql、Positioning、Location。

### Qt Creator 构建

1. 启动 Qt Creator，打开项目根目录 `CMakeLists.txt`。
2. 选择 `Desktop Qt 6.11.1 MSVC2022 64bit` Kit。
3. 点击 Configure Project、Build Project、Run。

### 命令行构建示例

```powershell
cmake -S . -B build-release -G Ninja `
  -DCMAKE_PREFIX_PATH="E:/Qt/6.11.1/msvc2022_64" `
  -DCMAKE_BUILD_TYPE=Release
cmake --build build-release
```

Qt 路径按新设备安装位置修改。源码迁移不依赖旧电脑构建路径；不要复制旧 `build`，应在新设备重新生成。

## 4. 页面操作

### 4.1 登录

当前演示账号为 `admin`，密码为 `123456`。登录成功后调用 `historyDatabase.startSession(username)` 创建本次历史会话。注册界面目前只做前端校验，不会永久保存用户，不应作为正式身份认证系统。

### 4.2 手机主页

主页左侧展示状态、海拔、速度、电量、经纬度、温湿度、姿态、曲线和告警，右侧为地图。

1. 让电脑和手机处于同一 Wi-Fi/热点网络。
2. 启动地面站，读取顶部显示的 URL，例如 `http://192.168.x.x:8080/`。
3. 手机浏览器打开 URL 并允许定位/传感器权限。
4. 手机开始上传后，Qt 实时更新数据与轨迹。

位置防抖会拒绝越界、零点、精度过差或速度异常的坐标，并根据定位精度设置约 3～12 m 静止死区，避免手机不动时图标原地旋转。

### 4.3 STM32 页面

1. 电脑连接 STM32 AP 热点。
2. 程序读取 WLAN IPv4 和掩码，将当前子网第一个主机地址作为 AP 网关。
3. Qt 以 TCP 客户端连接网关 `8080`。
4. 收到有效遥测后更新参数、地图、曲线和数据库。

| 按钮 | `cmd` | 作用 |
|---|---|---|
| 自由飞行 | `free_flight` | 自主模式，执行固件预设航线 |
| 起飞 | `takeoff` | 人工模式并执行起飞 |
| 悬停 | `hover` | 悬停 |
| 盘旋 | `loiter` | 按固件逻辑盘旋 |
| 改变航向 | `set_heading` | 使用页面设置航向角 |
| 返航 | `rth` | 返回返航点 |
| 降落 | `land` | 执行降落 |

按钮仅在 TCP 已连接且没有等待中的命令时可用。下发后最多等待 3 秒 ACK；超时只解除等待，不代表 STM32 已执行成功。

### 4.4 历史数据

点击主页顶部“历史数据”：

- 按登录会话展示手机/STM32 记录。
- 查看开始结束时间、样本数、起终点、电量和告警数。
- 查看轨迹、坐标、海拔、温湿度等曲线。
- 根据阈值列出异常点时间、坐标和原因。

## 5. 手机 HTTP 接口

Qt 在 IPv4 `8080` 端口监听：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/`、`/index.html` | 返回内嵌手机页面 |
| GET | `/health` | 服务状态和访问 URL |
| POST | `/gps` | 上传 GPS、姿态和电量 |
| POST | `/sensor` | 上传温湿度 |
| OPTIONS | 任意 | CORS 预检 |

GPS JSON：

```json
{
  "latitude": 40.113789,
  "longitude": 116.384795,
  "height": 120.0,
  "battery": 82,
  "pitch": 1.2,
  "roll": -0.8,
  "yaw": 135.0,
  "accuracy": 8.0
}
```

温湿度 JSON：

```json
{"temperature":24.6,"humidity":53.2}
```

成功返回 `{"ok":true}`，错误 JSON 返回 HTTP 400。

## 6. STM32 TCP/JSON 协议

### 6.1 角色和帧格式

- STM32/ESP8266：AP 和 TCP 服务端。
- Qt：TCP 客户端。
- 默认端口：`8080`。
- UTF-8 JSON，每帧以 `\r\n` 结束。

### 6.2 遥测帧

```json
{
  "id":"UAV_01",
  "proto":1,
  "seq":490,
  "lat":40.113789,
  "lng":116.384795,
  "heading":135.0,
  "target_alt":120,
  "speed":8.5,
  "rth_lat":40.113000,
  "rth_lng":116.384000,
  "rth_distance":95.2,
  "dht_temp":25,
  "dht_hum":52,
  "sht_temp":24.8,
  "sht_hum":51.6,
  "mcu_temp":42.1,
  "mode":0,
  "alarm":0,
  "battery":78,
  "ts":0
}
```

所有字段必填，数值必须是 JSON number，不能是字符串。`lat/lng` 为当前位置；`rth_lat/rth_lng` 为返航点；速度单位 m/s，距离/高度单位 m，电量范围 0～100。无 RTC 时 `ts` 可为 0。

程序支持 TCP 粘包/拆包，缓存上限 64 KiB。有效连接 3.5 秒没有遥测会重连；连接超时 3.5 秒，失败后约 2 秒重试。

### 6.3 Qt 下发命令

```json
{
  "type":"flight_command",
  "proto":1,
  "id":"UAV_01",
  "command_seq":123456,
  "cmd":"takeoff",
  "control":"manual",
  "use_preset_route":false,
  "heading":135,
  "target_alt":120,
  "speed":8,
  "ts":1780000000000
}
```

参数限制：`heading` 为 0≤x<360°，`target_alt` 为 0～500 m，`speed` 为 0～50 m/s。`free_flight` 使用 `control=autonomous`、`use_preset_route=true`；其他命令使用人工模式。

### 6.4 STM32 命令确认

```json
{
  "type":"command_ack",
  "command_seq":123456,
  "ok":true,
  "message":"accepted"
}
```

`type` 也允许 `ack`。`command_seq` 必须与 Qt 下发值一致；缺少序号或旧序号 ACK 会被忽略，避免旧 TCP 缓冲错误确认新命令。

## 7. 单设备互斥

手机 HTTP 是短连接，因此使用 8 秒活动租约：

1. 手机访问页面、健康接口或成功上传数据时标记活动。
2. 立即停止 STM32 重试、连接超时和遥测看门狗，断开已有 STM32 socket。
3. 手机每次有效请求刷新 8 秒计时器。
4. 8 秒无请求后判定手机离线并恢复 STM32 自动发现。

## 8. C++ 类和函数

### 8.1 `Backend`

负责手机 HTTP 服务、选择本机热点地址、解析手机 JSON 和 GPS 防抖。

| 函数 | 用途 |
|---|---|
| `Backend()` | 监听 8080，初始化手机活动计时器 |
| `preferredHotspotAddress()` | 从活动网卡选择手机可访问 IPv4 |
| `newConnection()` | 接收 HTTP TCP 连接 |
| `readData()` | 组装 HTTP 请求并路由 |
| `processGpsPayload()` | 校验 GPS、过滤漂移/跳点、更新位置姿态 |
| `processSensorPayload()` | 校验温湿度并发送信号 |
| `markPhoneActivity()` | 建立/刷新 8 秒手机租约 |
| `sendResponse()` | 生成 HTTP/CORS 响应 |

QML 属性包括 `latitude`、`longitude`、`height`、`speed`、`battery`、`pitch`、`roll`、`yaw`、`accuracy`、`accessUrl`、`phoneConnected`。

### 8.2 `SensorBackend`

负责手机温湿度、STM32 AP 发现、TCP 重连、遥测解析、看门狗、命令和 ACK。

| 函数 | 用途 |
|---|---|
| `discoverApServerAddress()` | 根据 WLAN 地址/掩码推导网关，不写死 IP |
| `attemptConnection()` | 选择目标并连接 |
| `reconnectTelemetry()` | 立即重连 |
| `connectToTelemetry(host, port)` | 使用手动目标 |
| `useAutomaticTelemetryTarget()` | 恢复自动发现 |
| `readTelemetryData()` | 处理粘包、拆包、缓存上限 |
| `processTelemetryFrame()` | 校验遥测或 ACK 并更新属性 |
| `sendFlightCommand()` | 校验参数、生成 JSON、等待 ACK |
| `setPhoneDeviceActive()` | 手机活动时暂停 STM32 |
| `updateSensorData()` | 保存手机温湿度 |

### 8.3 `HistoryDatabase`

负责 SQLite 建库、会话、记录、查询和异常识别。

| 函数 | 用途 |
|---|---|
| `portableProjectDirectory()` | 从构建目录定位项目根，部署时使用 EXE 目录 |
| `openDatabase()` | 打开 SQLite，启用 WAL/NORMAL/foreign_keys |
| `createTables()` | 创建两张数据表和索引 |
| `startSession(username)` | 创建时间戳+UUID 会话 ID |
| `recordPhoneGps()` | GPS 更新时写手机快照 |
| `recordPhoneSensor()` | 温湿度更新时写手机快照 |
| `recordStm32Telemetry()` | 写有效 STM32 遥测 |
| `getHistoryRecords()` | 返回会话摘要 |
| `getSessionData()` | 返回会话明细 |
| `getAbnormalData()` | 返回异常时间、坐标和原因 |

## 10. SQLite 数据库

路径为 `<项目>/database/FlightHistory.db`。开发运行时程序从 `<项目>/build/<kit>` 向上定位同时含 `CMakeLists.txt` 和 `historydatabase.cpp` 的项目根；独立部署时则使用 EXE 旁的 `database`。

- `phone_data`：会话、用户、时间、事件、坐标、海拔、速度、电量、姿态、精度、温湿度。
- `stm32_data`：会话、用户、设备、协议、序号、坐标、航向、高度、速度、返航信息、DHT/SHT、MCU 温度、模式、告警、电量和设备时间戳。

迁移数据库前必须完全退出程序。若存在 `.db-wal`、`.db-shm`，应和 `.db` 一起复制。Git 默认忽略真实数据库，避免发布用户名和位置数据；新克隆项目首次运行会自动建空库。

## 11. 异常阈值

### 手机

| 指标 | 异常条件 |
|---|---|
| 温度 | `< -20 ℃` 或 `> 40 ℃` |
| 湿度 | `> 90 %` |
| 电量 | 有效值 `< 20 %` |
| 速度 | `> 50 m/s` |
| 高度 | `< -100 m` 或 `> 500 m` |
| 俯仰/横滚 | 绝对值 `> 50°` |
| GPS 精度 | `> 50 m` |

### STM32

| 指标 | 异常条件 |
|---|---|
| DHT/SHT 温度 | `< -20 ℃` 或 `> 50 ℃` |
| DHT/SHT 湿度 | `< 10 %` 或 `> 90 %` |
| MCU 温度 | `< -20 ℃` 或 `> 75 ℃` |
| 电量 | `< 20 %` |
| 速度 | `> 50 m/s` |
| 目标高度 | `< 0 m` 或 `> 500 m` |
| 返航距离 | `> 2000 m` |
| 固件告警 | `alarm != 0` |

异常识别不会删除原始数据，只在历史告警视图附加原因。

## 12. 地图

- 使用 Qt Location OSM 插件。
- 支持鼠标/触控拖动、滚轮/手势缩放和跟随模式。
- 在线瓦片依赖网络。STM32 AP 没有互联网时，未缓存区域可能没有底图，但遥测轨迹仍可更新。

## 13. 排错

### 手机页面打不开

- 确认手机与电脑同网段，使用界面实时 URL，不要固定旧 IP。
- 防火墙允许程序监听 8080。
- 打开 `http://电脑IP:8080/health`，应返回 `{"ok":true,...}`。

### GPS 没反应

- 检查浏览器定位权限。
- 室内精度差时坐标可能被程序拒绝。
- 部分传感器 API 要求 HTTPS，具体以手机浏览器策略为准。

### STM32 热点已连但无数据

- STM32 必须是 TCP 服务端，Qt 是客户端。
- 服务端监听网关 `8080`，每帧完整 JSON 并以 `\r\n` 结束。
- 所有必填字段存在且数值不是字符串。
- 遥测间隔不要超过 3.5 秒。

### 命令被拒绝

- 检查 `cmd`、航向、高度和速度。
- ACK 携带正确 `command_seq`。
- STM32 应在同一连接持续发遥测、接收命令并回 ACK。

### 地图无底图

STM32 AP 通常无互联网，OSM 瓦片无法下载，这不等于遥测断开。可使用缓存、让 AP 提供外网或后续接入离线地图。

## 14. 发布与迁移

### 源码迁移

克隆仓库后安装 Qt 6.11.1/MSVC 2022，重新配置并构建，不复制旧 `build`。

### 无 Qt 环境运行

使用 Release 构建并执行 `windeployqt`，部署 Qt Core/Gui/Qml/Quick/Network/Sql/Positioning/Location、`platforms`、`sqldrivers/qsqlite`、`geoservices`、`position`、QML 模块、MSVC Release 运行库以及 EXE 旁的 `database` 目录。Debug 构建不适合作为便携发布包。
